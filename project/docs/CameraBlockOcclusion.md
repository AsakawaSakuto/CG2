# Camera Block Occlusion System

## 概要
プレイヤーとカメラの間にブロックがある場合、プレイヤーが隠れてしまう問題を解決するためのカメラ距離調整システムを実装しました。

## 実装内容

### 1. 機能説明
- カメラとプレイヤーの間にブロックがあるかをレイキャストでチェック
- ブロックが検出された場合、カメラをプレイヤーに近づける
- プレイヤーとブロックの距離に応じて、カメラの位置を自然に調整
- 遮るものがない場合は、元の距離にスムーズに戻る

### 2. 実装方法

#### GameCameraController.h の変更
- `CheckBlockOcclusion(Map3D* map)` メソッドを追加
- `Map3D` クラスの前方宣言を追加
- `originalDistance_` と `adjustedDistance_` を追加（元の距離と調整後の距離を管理）
- デバッグ用の `showDebugRay_` フラグを追加

#### GameCameraController.cpp の変更
- `CheckBlockOcclusion` メソッドを実装
  - プレイヤーからカメラへの方向ベクトルを計算
  - レイキャストでブロックとの交差をチェック
  - 最も近いブロックまでの距離を計算
  - ブロックの手前でカメラを停止（マージン 2.0f）
  - 最小距離は 5.0f に設定（プレイヤーに近づきすぎない）
  - 遮るものがない場合は、元の距離にスムーズに戻る（lerp速度 0.1）
- デバッグ表示機能
  - 遮られているブロックを赤いAABBで表示
  - カメラとプレイヤー間のレイを色分けして表示
    - 緑：遮られていない
    - 黄：遮られている（調整後の位置まで）

#### GameScene.cpp の変更
- `Update()` メソッド内で `gameCamera_->CheckBlockOcclusion(map3D_.get())` を呼び出し
- カメラ更新の直後に配置

### 3. アルゴリズム詳細

```
1. プレイヤー位置（targetPosition_ + offset_）を取得
2. カメラの理想的な位置を取得
3. プレイヤーからカメラへの方向ベクトルを計算
4. 方向ベクトルを正規化
5. レイキャストのステップ数を計算（1.0f間隔）
6. 各ステップでチェック：
   - ワールド座標→マップ座標に変換
   - TileTypeがNormalかチェック
   - ブロックのAABBとの交差判定
   - 交差している場合、距離を記録
7. 最も近いブロックが見つかった場合：
   - adjustedDistance = blockDistance - margin
   - 最小距離（5.0f）でクランプ
8. 見つからなかった場合：
   - adjustedDistance を originalDistance に向けて lerp
```

### 4. パラメータ

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| stepSize | fullDistance / steps | レイキャストのステップ間隔（約1.0f） |
| margin | 2.0f | ブロックとカメラの安全マージン |
| minDistance | 5.0f | カメラの最小距離（プレイヤーに近づきすぎない） |
| lerpSpeed | 0.1f | 元の距離に戻る速度 |

### 5. 動作例

#### 例1: ブロックがプレイヤーの近くにある場合
```
Player: {10, 0, 0}
Block:  {5, 0, 0}
Camera: {0, 0, 0}

ブロックとプレイヤーの距離: 5.0
→ adjustedDistance ? 5.0（minDistanceでクランプ）
→ カメラはブロックの手前（約{7.0, 0, 0}付近）に配置
```

#### 例2: ブロックがプレイヤーから遠い場合
```
Player: {15, 0, 0}
Block:  {5, 0, 0}
Camera: {0, 0, 0}

ブロックとプレイヤーの距離: 10.0
→ adjustedDistance ? 8.0（margin分を引く）
→ カメラはブロックの手前（約{7.0, 0, 0}付近）に配置
```

### 6. デバッグ機能
- ImGuiで `Show Debug Ray` をオン/オフ可能
- 遮られているブロックが赤いAABBで表示される
- カメラとプレイヤー間のレイが表示される
  - 緑：正常（遮られていない）
  - 黄：遮られている（調整後の位置）

### 7. 今後の改善案
1. より正確なレイとAABBの交差判定
2. スロープブロックも考慮する
3. カメラの回転に応じた最適化
4. 複数のブロックが重なる場合の処理改善
5. カメラの移動をよりスムーズにする（イージング関数の適用）

## 使用方法

### コード例
```cpp
// GameScene::Update() 内
gameCamera_->SetTarget(player_->GetPosition());
gameCamera_->Update();

// ブロック遮蔽のチェックと距離調整
gameCamera_->CheckBlockOcclusion(map3D_.get());
```

### デバッグ表示
```cpp
// ImGui でデバッグ情報を表示
gameCamera_->DrawImgui();
// "Show Debug Ray" をチェックすると、レイとブロックが表示される
```

## まとめ
この実装により、プレイヤーとカメラの間にブロックがある場合でも、プレイヤーが常に見えるようにカメラが自動的に調整されます。プレイヤーとブロックの距離に応じて、カメラの位置も自然に変化します。
